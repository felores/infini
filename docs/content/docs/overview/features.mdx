# Features

This document covers the main features currently implemented in the project.

## Canvas Projects

- Support for multiple canvas projects.
- Rename, delete, batch select, and batch delete projects.
- Export a single canvas project as JSON, and import canvases from JSON.
- Canvas projects are stored in the browser locally; logging in does not auto-sync to the server.

## Infinite Canvas

- Drag canvas, scroll-wheel zoom, zoom slider, and reset view.
- Minimap positioning with toggle.
- Three background modes: dots, grid lines, and blank.
- Light and dark themes.
- Box select, multi-select, select all, deselect, and delete selected.
- Copy/paste nodes and connections between nodes.
- Undo and redo for nodes, connections, viewport, background, and assistant sessions.
- Node connections with upstream/downstream highlight.
- Keyboard shortcut help covering zoom, box select, select all, copy/paste, undo/redo, delete, deselect, and drag-in images.

## Nodes

There are currently three main node types on the canvas:

- **Image node**: Displays uploaded images, generated images, or My Assets images.
- **Text node**: Saves prompts, descriptions, AI text responses, and other text content.
- **Generation config node**: Aggregates upstream text and images to batch-generate images or text with unified model, ratio, and count settings.

Nodes support:

- Drag to move.
- Four-corner resize.
- Proportional or free-ratio scaling for image nodes.
- View basic node info and JSON.
- Delete, copy, paste.
- Establish upstream/downstream relationships via left/right connection points.

## Image Workflow

- Upload images to new nodes.
- Drag image files onto the canvas.
- Replace existing image node content.
- Download image nodes.
- Save image nodes to "My Assets".
- Crop images and generate the result as a new image node.
- Local multi-angle transform and generate the result as a new image node.
- Retry after generation failure.
- Batch-generating multiple images displays as an image group node with stacked card preview, expand to view all results, and set main image.

## AI Generation

The project uses direct frontend connections by default: the browser uses locally configured Base URL, API Key, and Model to call OpenAI-compatible APIs directly, without routing through a project backend.

OpenAI-compatible image and text capabilities continue using existing endpoints:

- `/v1/images/generations`: Text-to-image.
- `/v1/images/edits`: Image-to-image / reference image editing.
- `/v1/responses`: Text Q&A, image Q&A, and online Agent tool calls.
- `/v1/models`: Fetch model list; Volcano Ark Agent Plan's `/api/plan/v3` does not currently provide an OpenAI `/models` endpoint; model names must be entered manually.

Video capabilities support two API types:

- OpenAI-style video: `POST /v1/videos`, `GET /v1/videos/{id}`, `GET /v1/videos/{id}/content`.
- Volcano Ark Agent Plan / Seedance 2.0: Base URL uses `https://ark.cn-beijing.volces.com/api/plan/v3`, creates tasks via `POST /contents/generations/tasks`, queries tasks via `GET /contents/generations/tasks/{id}`, reads `content.video_url` on success.

If the Base URL already ends with `/v1`, `/api/v3`, or `/api/plan/v3`, the system will not append `/v1`. Therefore, CPA reverse proxy or Volcano Ark Agent Plan can continue to be configured via existing Base URL + API Key + Model without adding a new provider.

The "Fetch model list" button in the config modal makes a real request to OpenAI `/models` and does not fabricate results for Agent Plan. If Volcano Ark Agent Plan returns 404, manually add `doubao-seedance-2.0` or other model names listed in the docs.

Configurable options:

- Base URL.
- API Key.
- Default model.
- Image quality.
- Image ratio.
- Generation count.
- Video model.
- Video ratio, resolution, duration, audio generation, and watermark.

Regular image/text nodes can directly input a prompt and generate results. Generation config nodes can read upstream node content and batch-generate multiple image or text results based on the node's own configuration. Generation config nodes support previewing the current prompt and reference image inputs, and adjusting input order.

Video generation can read prompts from text nodes, reference images from image nodes, reference videos from video nodes, and reference audio from audio nodes. Seedance 2.0 supports up to 9 reference images, 3 reference videos, and 3 reference audio clips; resolutions: `480p`, `720p`, `1080p` (fast model does not support `1080p`); ratios: `16:9`, `4:3`, `1:1`, `3:4`, `9:16`, `21:9`, `adaptive`; durations: 4-15 seconds or smart duration. On success, the video is inserted into the canvas as a video node with a native player preview. Reference videos and audio prefer publicly accessible URLs; local assets are passed as frontend-readable data to the compatible API, and support depends on the specific upstream.

## Canvas Assistant

The assistant panel on the right side of the canvas supports:

- Text Q&A.
- Image generation.
- Reading the currently selected node as a reference.
- Automatically including upstream nodes of the selected node as references.
- Pasting images into the assistant input and inserting them onto the canvas.
- Conversation history.
- Deleting individual or multiple conversations.
- Retrying responses.
- Inserting assistant-generated text onto the canvas.
- Inserting assistant-generated images onto the canvas.
- Collapsing and expanding the assistant panel.

## Prompt Library

The frontend prompt library supports:

- Search by title.
- Filter by tags.
- Filter by source.
- View prompt details.
- View cover and result images.
- Copy prompts.
- Add prompts to "My Assets".

Prompt management supports:

- Query prompts.
- Filter by group and tags.
- View remote prompt sources.
- Trigger fetching built-in remote prompt sources.

Currently, built-in remote sources include multiple GPT Image / GPT-4o / Nano Banana Pro related prompt repositories, fetched directly by the browser frontend and cached in IndexedDB.

## Assets

"My Assets" is a browser-local asset library that supports:

- Adding text and image assets.
- Editing asset title, cover, tags, source, note, and content.
- Deleting assets.
- Search by keyword.
- Filter by type.
- Paginated browsing.
- Copy text assets.
- Download image assets.
- Add assets from the prompt library and canvas nodes.
- Insert assets onto the canvas.

## Configuration and Sync

- The current version does not require account login and does not provide an admin backend.
- The config and preferences modal supports multiple OpenAI-compatible channels, default models, generation preferences, and WebDAV sync settings.
- Third-party prompts, WebDAV, and AI APIs are all connected directly from the browser frontend without a project backend proxy.

## Current Limitations

- Canvas projects and "My Assets" are currently only stored in the browser locally and do not sync with accounts.
- The AI API Key is stored in the browser and used to directly call the configured OpenAI-compatible endpoint; suitable only for personal or trusted environments.
- Seedance local reference videos/audio are best served via publicly accessible URLs; whether the upstream accepts locally-passed data depends on the specific compatible API.
- When Seedance returns a remote video URL, the frontend tries to download it as a local Blob for persistence; if CORS or network restrictions prevent download, the remote URL is retained, and subsequent playback depends on the upstream URL's expiry.
- The canvas is designed primarily for desktop use; mobile touch experience has not been systematically refined.
