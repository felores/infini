# Docker Deploy

If you want to run the project on your own machine or server, you can use Docker Compose directly.

## Using Published Image

```bash
git clone git@github.com:basketikun/infinite-canvas.git
cd infinite-canvas
docker compose up -d
```

After starting, visit:

```text
http://localhost:3000
```

## Building Image Locally

If you need to build an image from the current source:

```bash
docker compose -f docker-compose.local.yml up -d --build
```

## Docs Site Image

The docs site is in the `docs/` directory, built separately as a Next.js standalone app with server capabilities. It is not included in the main app image.

Using published image:

```bash
cd docs
docker compose up -d
```

Building from current source:

```bash
cd docs
docker compose -f docker-compose.local.yml up -d --build
```

## Data Notes

The main app image only starts Next.js. Canvas data, My Assets, generation logs, and the AI API Key are stored in the browser by default; third-party prompts are fetched directly by the browser frontend, so no additional data volume mounting is needed.
