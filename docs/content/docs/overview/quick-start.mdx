# Quick Start

If you just want to get the project running, prioritize deploying or starting the `web/` frontend.

## Vercel Deploy

Import the repository in Vercel. The root `vercel.json` will build `web/`. In the current version, AI requests are made directly from the browser to the user's own OpenAI-compatible endpoint; no server-side configuration is needed.

## Local Startup

```bash
git clone git@github.com:basketikun/infinite-canvas.git
cd infinite-canvas
cd web
bun install
bun run dev
```

After starting, visit:

```text
http://localhost:3000
```

## Docker Startup

If you need to build an image from the current source:

```bash
docker build -t infinite-canvas .
docker run --rm -p 3000:3000 infinite-canvas
```

## First-Time Tips

- Open the config modal in the top-right corner and enter your `Base URL`, `API Key`, and model name.
- For prompt library content, opening `/prompts` will fetch and cache it in IndexedDB directly from the browser.
- For cross-device sync of canvases, assets, and generation logs, configure WebDAV in the config modal.

## Notes

- Canvas projects and "My Assets" are stored primarily in the browser locally; WebDAV sync requires user configuration.
- The AI API Key is stored in the browser and used by the frontend to call OpenAI-compatible APIs directly.
