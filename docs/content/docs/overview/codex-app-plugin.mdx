# Codex App Plugin

Infinite Canvas provides a Codex App plugin to connect Codex with the local Canvas Agent. After installation, you can say "Open Infinite Canvas" directly in Codex to have it open the canvas, read nodes, create nodes, and trigger generation flows.

The plugin source is in the repository's `plugins/infinite-canvas/` directory.

## Installation Methods

### Let Codex Auto-Install

Send the following to Codex:

```text
Please install the Infinite Canvas Codex plugin from https://github.com/basketikun/infinite-canvas.git.
Clone the repository to ~/plugins/infinite-canvas, confirm plugins/infinite-canvas/.codex-plugin/plugin.json exists,
add plugins/infinite-canvas to the personal marketplace, first run codex plugin marketplace add ~,
then run codex plugin add infinite-canvas@personal.
After installation, verify the plugin and tell me whether I need to start a new conversation to load the new skill and MCP tools.
```

This method is suitable for users who don't want to manually edit marketplace config. Codex handles cloning, writing marketplace config, installing the plugin, and basic verification.

### Manual Install

Recommended: clone the repository to the location Codex's personal marketplace references by default:

```bash
mkdir -p ~/plugins
git clone https://github.com/basketikun/infinite-canvas.git ~/plugins/infinite-canvas
```

Ensure `~/.agents/plugins/marketplace.json` has an Infinite Canvas entry. The `path` must point to the plugin subdirectory inside the repo:

```json
{
  "name": "personal",
  "interface": {
    "displayName": "Personal"
  },
  "plugins": [
    {
      "name": "infinite-canvas",
      "source": {
        "source": "local",
        "path": "./plugins/infinite-canvas/plugins/infinite-canvas"
      },
      "policy": {
        "installation": "AVAILABLE",
        "authentication": "ON_INSTALL"
      },
      "category": "Productivity"
    }
  ]
}
```

Then register the personal marketplace and install the plugin:

```bash
codex plugin marketplace add ~
codex plugin add infinite-canvas@personal
```

After installation, start a new Codex conversation so the new skill and MCP tools load fully.

### Dev Debug Install

If you are developing the plugin within the Infinite Canvas repo, use the repo's own marketplace:

```bash
cd /path/to/infinite-canvas
codex plugin marketplace add "$(pwd)"
codex plugin add infinite-canvas@infinite-canvas-local
```

The repo's `.agents/plugins/marketplace.json` already points to `./plugins/infinite-canvas`, suitable for local development debugging.

## Using the Plugin

After starting a new Codex conversation, you can say:

```text
Open Infinite Canvas, create a new canvas, and connect Codex
```

The plugin first checks whether the local canvas service for the current repo is running. If the port is occupied, it checks whether the listening process belongs to the current project to avoid connecting to another project. After confirming or starting the canvas, the plugin starts the local Canvas Agent and opens the new canvas URL with connection parameters.

Once the canvas is open, you can continue having Codex operate on the canvas:

```text
Read the current canvas and summarize the node structure
Create a set of image generation prompts based on selected nodes
Organize these nodes into a generation flow
```

## Updating the Plugin

Regular code changes usually don't require updating the plugin. Only reinstall or refresh the plugin when changing:

- `plugins/infinite-canvas/.codex-plugin/plugin.json`
- `plugins/infinite-canvas/.mcp.json`
- Skill descriptions under `plugins/infinite-canvas/skills/`
- Plugin-exposed MCP, app, or entry scripts

After reinstalling, start a new conversation to avoid old conversations using cached skills and MCP tools.

## Manual Troubleshooting

The plugin starts the Canvas Agent via the following MCP command by default:

```bash
npx -y @basketikun/canvas-agent@0.1.0 mcp
```

For manual troubleshooting, start the canvas first:

```bash
cd web
bun install
bun run dev
```

Then start the local Agent. If the port is not `3000`, replace `CANVAS_URL` with the real local canvas address:

```bash
CANVAS_URL=http://localhost:3000 bun --cwd canvas-agent src/index.ts
```

After reading the local address and token from the Agent output or `http://127.0.0.1:17371/config`, open:

```text
<canvas-web-address>/canvas?mode=new#agentUrl=<Local URL>&agentToken=<Connect token>
```

`mode=new` makes the web page auto-create a specific canvas and connect to the local Agent without manually clicking "New Canvas".
