# Canvas Node Manual

This document covers the main uses and operation flows for canvas nodes, for easy use and ongoing development. Currently covers text nodes first.

## Text Nodes

Text nodes are used to save prompts, drafts, descriptions, and AI-generated text results. They can serve as standalone text content or as input for downstream generation config nodes.

### Editing Text Content

- Double-click the text node content area to edit the text directly.
- Select a text node, then click "Edit Text" in the top toolbar to enter text editing mode.
- Use "Zoom out" and "Zoom in" in the top toolbar to adjust the text node font size.

### Generating or Modifying Text via the Dialog Below

- Select a text node, then click "Edit" in the top toolbar to open the dialog below the node.
- The dialog below a text node is only for generating or rewriting text; it does not handle image generation.
- When the text node is empty, the input field is for entering the text you want to generate; after clicking send, the result fills the current text node.
- When the text node already has content, the input field is for entering what you want to change the text to; after clicking send, a new text node is generated on the right and automatically connected to the original node.
- Input can be hand-written or selected from the prompt library.
- The model dropdown in the dialog comes from the globally fetched model list; the selection only applies to the current node and does not modify other nodes or the global default model.
- If no models appear in the dropdown, open the config modal to fetch the model list and set default image and text models. If Volcano Ark Agent Plan reports `/models` as unsupported, enter model names manually.

### Generating Images from Text Nodes

- First prepare the text content for image generation in the text node.
- Click the "Generate Image" button in the text node's top toolbar.
- The system automatically creates a generation config node to the right of the text node and connects the text node to it.
- The generation config node reads the upstream text content as the image prompt and immediately starts generating images.
- To adjust model, ratio, or count later, modify the generation config node and regenerate.

### Continuing Image Generation from Image Nodes

- Select an empty image node and click "Generate" to generate a result directly in the current node, without keeping an extra empty placeholder.
- When count is `1`, the result fills the current image node directly.
- When count is greater than `1`, the current image node becomes the main image node, with the corresponding number of child image nodes generated to the right.
- If the image node already has content, the existing image is used as a reference image to generate a new batch.
- Each generated image node records the prompt, generation type, model, size, quality, count, and reference image references in its own `metadata` for later retries.
- If the reference image is lost or cannot be restored during retry, the system will notify directly to avoid sending empty image-to-image requests.

### Video Nodes

- The toolbar can create video nodes; you can also drag in or upload local video files.
- Video nodes use a native player for content display, with play, pause, and seek within the node.
- The dialog below an empty video node can accept a prompt to generate video; the result fills the current node.
- When creating video generation from text, image, or config nodes, a new video node is generated on the right and auto-connected.
- The video mode of the generation config node reads upstream text as the prompt, upstream images as reference images, and upstream videos as reference videos, displaying reference videos in the input preview.
- The video generation API supports OpenAI-style `POST /v1/videos`, `GET /v1/videos/{id}`, and `GET /v1/videos/{id}/content`.
- When using Volcano Ark Agent Plan / Seedance 2.0, set the Base URL to `https://ark.cn-beijing.volces.com/api/plan/v3` and use the Seedance 2.0 model name; the system switches to `POST /contents/generations/tasks` to create async tasks and polls `GET /contents/generations/tasks/{id}`.
- Agent Plan's `/api/plan/v3` does not currently provide an OpenAI `/models` model list endpoint; the config modal will not fabricate a model list. Please manually enter `doubao-seedance-2.0` or other available models listed in the docs.
- Seedance reference videos are best served via publicly accessible URLs; local assets are read by the frontend and passed to the compatible API, and availability depends on the specific upstream.

### Recommended Workflow

1. Create a new text node and write your image creation idea.
2. Open the dialog below the text node and let AI optimize or expand the prompt.
3. The rewritten result generates into a new text node, preserving the original text for comparison.
4. After confirming the text node content, click "Generate Image" in the top toolbar.
5. Continue adjusting image parameters or regenerate in the automatically created generation config node.
