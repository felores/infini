# Local Development

The main app is primarily the `web/` frontend. AI requests are made directly from the browser to the user's own OpenAI-compatible endpoint.

## 1. Start the Frontend

```bash
cd web
bun install
bun run dev
```

The frontend defaults to:

```text
http://localhost:3000
```

## 2. Configure Models

Open the config modal in the top-right corner and enter your `Base URL`, `API Key`, and model name. Third-party prompts and WebDAV are both connected directly from the browser frontend.

## 3. Start the Docs Site

If you need to adjust the docs site separately, run in the `docs` directory:

```bash
bun run dev
```

## Common Scenarios

- Modify canvas, pages, and interactions: mainly `web/`
- Modify prompt caching: mainly `web/src/services/api/prompts.ts`
- Modify docs site content: mainly `docs/content/docs/`
