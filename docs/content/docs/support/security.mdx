# Vulnerability Reporting

The English original security policy is at [SECURITY.md](https://github.com/basketikun/infinite-canvas/blob/main/SECURITY.md).

Do not post vulnerability details, exploitable code, private API keys, sensitive information in screenshots, or real user data in public Issues.

## Reporting Process

Prefer using GitHub's private vulnerability reporting or Security Advisory. If the repository does not have the corresponding entry enabled, send an email to [1844025705@qq.com](mailto:1844025705@qq.com) with the subject:

```text
[infinite-canvas security]
```

If you cannot use a private channel yet, open a public Issue requesting the maintainer to provide a private contact method, but do not write vulnerability details in the Issue.

## Recommended Information to Provide

- Affected version, commit, branch, or deployment method.
- Clear reproduction steps.
- Vulnerability impact, attack scenarios, and trigger conditions.
- Sanitized logs, screenshots, or PoC.
- Whether it involves browser local storage, API Key, WebDAV sync, AI channel configuration, or API requests.

## Scope

The project prioritizes XSS, sensitive information leakage, file handling, import/export, WebDAV sync, access control, and supply chain risks caused by this repository's code or default configuration.

Third-party model services, hosting platforms, browser extensions, user-leaked API keys, security header suggestions without practical exploit paths, pure social engineering phishing, or account recovery issues are generally not within this project's vulnerability scope.
